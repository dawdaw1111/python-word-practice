# 项目定位

**项目名：PyWord / Python单词拼写练习**

## 一句话介绍

一个专门给 Python 初学者用的英文单词拼写练习工具。
 用户双击网页就能打开，通过“看中文、拼英文”的方式记忆 Python 学习中最常见的单词，并自动保存学习进度。

------

# 一、MVP目标

这个 MVP 只做一件事：

**帮助 Python 小白用最简单的方式记住常见编程英文单词。**

第一版要验证的是：

1. 用户能不能直接打开就用
2. 用户愿不愿意通过拼写来记忆
3. 用户能不能持续使用并保留进度

所以第一版必须满足：

- 不需要安装
- 不需要联网
- 不需要注册
- 不需要框架
- 手机也能用
- 关闭后再打开还能继续

------

# 二、目标用户

## 核心用户

- Python 初学者
- 英语较弱的编程小白
- 培训班学员
- 刚开始接触 Python 的学生

## 核心痛点

- 看不懂 `print`、`input`、`list`、`string`
- 看到报错英文就慌
- 背普通单词软件太泛，不贴合 Python
- 不想折腾安装环境
- 想点开就能学

------

# 三、产品形式

这个项目最终形态就是一个**本地离线网页小工具**。

用户拿到后只需要：

1. 解压文件夹
2. 双击 `index.html`
3. 直接开始学习

如果是在手机端：

- 可以放到手机浏览器里打开
- 或部署成静态网页后直接访问

------

# 五、MVP核心功能

## 1. 首页

用户打开网页后先看到首页。

首页内容：

- 项目标题
- 简短说明
- 开始学习按钮
- 继续学习按钮
- 当前学习进度摘要
- 进入错题本按钮
- 查看统计按钮

### 首页文案示例

标题：
 **Python单词拼写练习**

副标题：
 **专为 Python 初学者设计，边学编程边记英文。**

按钮：

- 开始学习
- 继续学习
- 错题复习
- 学习统计

进度区：

- 已学：15 个
- 已掌握：9 个
- 错题：4 个
- 当前分类：基础入门

------

## 2. 学习页

这是最核心的页面。

每次展示 1 个单词题目。

### 页面内容

- 当前分类
- 当前题号
- 中文意思
- 场景提示
- 输入框
- 提交按钮
- 跳过按钮
- 当前进度条

### 示例

分类：基础入门
 第 3 / 15 题

中文：**打印；输出**
 提示：**把内容显示到屏幕上**

输入框：`___________`

按钮：

- 提交
- 跳过

------

## 3. 判题反馈

用户提交后，立即显示结果。

### 如果答对

显示：

- 回答正确
- 正确单词
- 简短说明
- 示例代码
- 下一题按钮

例如：

**回答正确！**
 单词：`print`
 说明：用于把内容输出到屏幕
 示例：

```
print("Hello")
```

### 如果答错

显示：

- 回答错误
- 正确答案
- 你的答案
- 示例代码
- 自动加入错题本

例如：

**回答错误**
 你的答案：`pritn`
 正确答案：`print`

------

## 4. 继续学习

用户关闭网页后，下次再打开，点击“继续学习”可以回到上次位置。

需要保存：

- 当前分类
- 当前题目索引
- 当前已完成进度

------

## 5. 错题本

答错的单词自动进入错题本。

错题本至少显示：

- 中文
- 正确英文
- 错误次数
- 再练一次按钮

还可以有：

- 清空已掌握错题
- 开始错题复习

------

## 6. 学习统计页

让用户看到自己的进步。

统计页最少显示：

- 总词数
- 已学词数
- 已掌握词数
- 错题数
- 当前分类
- 总正确率

------

## 7. 分类闯关

单词按模块分类，用户一组一组学。

建议第一版做这几个分类：

- 基础入门
- 数据类型
- 流程控制
- 函数基础
- 常见报错词

每类 10~15 个词就够了。

------

# 七、页面结构

整个项目建议只做 4 个页面，或者做成单页切换也行。

## 方案A：多页面

- `index.html` 首页
- `study.html` 学习页
- `wrong.html` 错题本
- `stats.html` 统计页

## 方案B：单页面切换

只保留一个 `index.html`，通过 JS 控制页面区域切换。

### 我更推荐你用方案B

因为：

- 文件少
- 双击就能用
- 更适合新手
- 管理简单
- 不容易跳转丢状态

# 十二、核心交互逻辑

## 1. 开始学习

- 没有历史记录：从第一类第一题开始
- 有历史记录：从当前进度开始

## 2. 提交答案

处理规则：

- 去除前后空格
- 忽略大小写
- 和标准答案比较

### 答对

- 更新正确次数
- 加入已学习
- 连续多次正确可加入已掌握
- 自动进入下一题

### 答错

- 更新错误次数
- 加入错题本
- 展示正确答案
- 再进入下一题

## 3. 掌握逻辑

第一版可以简单一点：

- 某个单词答对 2 次，算已掌握

# 十三、手机端需求怎么满足

这个项目完全可以做成**响应式页面**。

## 手机端必须满足的点

### 1. 大按钮

按钮高度至少 44px 以上，方便点击。

### 2. 大输入框

输入框足够高，手机键盘弹出后仍然好操作。

### 3. 单列布局

手机端不要左右两栏，全部改成纵向布局。

### 4. 字体清晰

正文建议 16px 起，标题 24px 左右。

### 5. 底部留白

避免手机浏览器底部遮挡按钮。

------

## 响应式建议

使用 CSS 媒体查询：

```
@media (max-width: 768px) {
  .container {
    width: 92%;
    padding: 16px;
  }

  .main-card {
    padding: 20px;
    border-radius: 16px;
  }

  .btn {
    width: 100%;
    margin-bottom: 12px;
  }
}
```
