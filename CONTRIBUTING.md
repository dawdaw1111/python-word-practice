# Contributing to PyWord Lite

Thanks for your interest in contributing.

## Workflow

1. Fork this repository.
2. Create a feature branch from main.
3. Keep commits focused and atomic.
4. Open a Pull Request with a clear summary.

## Pull Request Checklist

- [ ] Code runs locally without errors
- [ ] README/docs are updated when behavior changes
- [ ] New files use clear names and consistent style
- [ ] No secrets or local environment files are committed

## Commit Message Convention

Use a concise prefix style:

- eat: new feature
- ix: bug fix
- docs: documentation update
- efactor: code refactor without behavior change
- chore: tooling or maintenance

Example:

`	ext
feat: add category filter on practice page
`

## Code Style

- Keep functions small and readable
- Prefer explicit naming over short abbreviations
- Avoid unnecessary dependencies
- Preserve existing project structure and conventions

## Reporting Issues

Please include:

- Steps to reproduce
- Expected behavior
- Actual behavior
- Browser / runtime version
- Screenshot (if UI related)
